Comentarios sobre Octave¶

  • Para la interpolaci√≥n de Hermite debemos utilizar variables de tipo celda (cell). Las celdas se pueden definir directamente como los vectores o matrices, pero utilizando llaves ({ }) en lugar de corchetes ([ ]). As√≠, por ejemplo, si escribimos

y={[1,2],[0,1,1],[3],[1,1]}

estar√≠amos definiendo una celda con una fila y cuatro ¬´columnas¬ª (denominadas y{1,1}, y{1,2}, etc.), cada una de ellas contiene los vectores indicados entre las llaves (que son vectores fila en realidad, por ejemplo y{1,2} es de tama√±o 1 √ó 3). Como ves podemos de esta forma definir objetos estructurados de tipo matriz, pero donde la longitud de las columnas (o filas) es distinto para cada fila (o columna); incluso podemos definir una componente de una celda como una matriz o una celda.

En Octave no es posible indexar sobre las componentes de una celda. Por ejemplo, con la definici√≥n anterior de la celda y, no podemos hacer y{1:4}(1), que ser√≠a lo natural para recuperar la primera componente de cada una de las cuatro filas de y. S√≠ podemos indexar como y{2:4}, que devuelve las componentes 2 a 4 (que ser√≠a m√°s l√≥gico y v√°lido denominar y{:,2:4}). Tambi√©n podemos recuperar una fila, por ejemplo, con z=y{2} obtenemos un vector z cuyo valor es [0,1,1]. Para recuperar por ejemplo la primera componente de cada componente haremos un bucle:

In [6]:
%{
for i=1:length(y)
    z(i) = y{i}(1)
endfor
%}
  • A menudo las ventanas gr√°ficas quedan escondidas tras otras ventanas (si no sueles trabajar a la vez con varias ventanas, esto no es un problema) o tienen un tama√±o o proporciones inadecuadas. Un truco para hacer aparecer las figuras previstas en un script puede ser incluir todas las √≥rdenes que generan las figuras al final del script, pero esto obviamente puede ser molesto.

Una forma de hacerlas visibles es decidir d√≥nde se van a colocar en pantalla y qu√© tama√±o tendr√°n. Para esto disponemos de la orden siguiente:
